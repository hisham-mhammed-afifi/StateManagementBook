# Research: Selectors and Memoization

**Date:** 2026-03-30
**Chapter:** Ch 9
**Status:** Ready for chapter generation

## API Surface

### `createSelector` (Stable)
- **Import:** `@ngrx/store`
- **Signature:** Supports 1-8 input selectors + projector function. Returns `MemoizedSelector<State, Result, ProjectorFn>`.
- Also supports dictionary-based overload (no projector):
  ```typescript
  const selectViewModel = createSelector({
    users: selectUsers,
    loading: selectLoading,
    error: selectError,
  });
  // Returns { users: User[], loading: boolean, error: string | null }
  ```

### `createFeatureSelector` (Stable)
- **Import:** `@ngrx/store`
- **Signature:** `createFeatureSelector<T>(featureName: string): MemoizedSelector<object, T>`
- **Deprecated overload:** `createFeatureSelector<T, V>(featureName: keyof T)` -- use single-generic version instead.

### `createSelectorFactory` (Stable)
- **Import:** `@ngrx/store`
- **Signature:** `createSelectorFactory(memoize: MemoizeFn, options?: SelectorFactoryConfig): (...input: any[]) => MemoizedSelector`
- Lower-level API that `createSelector` wraps. Accepts custom memoization function.

### `defaultMemoize` (Stable)
- **Import:** `@ngrx/store`
- **Signature:** `defaultMemoize(projectionFn: AnyFn, isArgumentsEqual?: ComparatorFn, isResultEqual?: ComparatorFn): MemoizedProjection`
- Last-1 cache (caches only the most recent result).
- `isArgumentsEqual` defaults to `===` (reference equality).
- `isResultEqual` is optional; prevents downstream emissions when result is semantically equal.

### `resultMemoize` (Stable)
- **Import:** `@ngrx/store`
- **Signature:** `resultMemoize(projectionFn: AnyFn, isResultEqual: ComparatorFn): MemoizedProjection`
- Convenience wrapper around `defaultMemoize` with a result comparator.

### `MemoizedSelector` interface (Stable)
- **Import:** `@ngrx/store`
- Key members:
  - `release()` -- clears memoized values recursively through ancestor selectors
  - `projector` -- the raw projector function (useful for unit testing without store)
  - `setResult(result?)` -- manually set cached result
  - `clearResult()` -- clear cached result

### `Store#selectSignal` (Stable since v17)
- **Signature:** `store.selectSignal(selector, options?: SelectSignalOptions): Signal<T>`
- `SelectSignalOptions` has `equal` property for custom equality checking.
- Preferred over `toSignal(store.select(...))` in Angular 21 zoneless mode.

### `Store#select` (Stable)
- **Signature:** `store.select(selector): Observable<T>`
- Returns Observable; works with RxJS pipe chains.
- Uses `distinctUntilChanged` internally.

### Entity Adapter Selectors (Stable, covered more in Ch 11)
- `adapter.getSelectors()` returns `{ selectIds, selectEntities, selectAll, selectTotal }`
- Can be scoped: `adapter.getSelectors(selectFeatureState)`

## Key Concepts

- **Selectors as the read layer:** Pure functions that extract and derive slices of state from the store.
- **Memoization:** `createSelector` caches the last result and only recomputes when input selector results change by reference (`===`).
- **Composition:** Build complex queries by composing simple selectors. Selector graph mirrors component needs.
- **No derived state in the store:** Never store computed values alongside raw data. Selectors compute derived state on-the-fly with memoization.
- **Granularity matters:** Fine-grained input selectors prevent unnecessary recomputation. Using a broad feature selector when only one property is needed wastes computation.
- **View model selectors:** One selector per smart component, returning a typed object with all data the component needs. Reduces multiple subscriptions to one.
- **Factory (parameterized) selectors:** Replace deprecated selectors-with-props (deprecated since v11). Each invocation creates a new selector instance.
- **Signal bridge:** `store.selectSignal()` returns `Signal<T>` for direct use in templates and `computed()`/`effect()` without async pipe.
- **`createFeature` auto-generation:** Chapter 8 covered `createFeature`; this chapter deepens selector composition via `extraSelectors`.
- **Selector layering strategy:**
  1. Base selectors (property access, auto-generated by `createFeature`)
  2. Derived selectors (combine base selectors + transform)
  3. View model selectors (component-specific shapes)
  4. Cross-feature selectors (multi-feature composition)

## Code Patterns

### Pattern 1: createFeatureSelector + createSelector (manual approach)
```typescript
// src/app/products/state/products.selectors.ts
import { createFeatureSelector, createSelector } from '@ngrx/store';
import { ProductsState } from './products.reducer';

export const selectProductsState = createFeatureSelector<ProductsState>('products');
export const selectProducts = createSelector(selectProductsState, state => state.products);
export const selectLoading = createSelector(selectProductsState, state => state.loading);
export const selectError = createSelector(selectProductsState, state => state.error);
```

### Pattern 2: Composed selectors
```typescript
// src/app/products/state/products.selectors.ts
export const selectAvailableProducts = createSelector(
  selectProducts,
  products => products.filter(p => p.inStock)
);

export const selectProductCount = createSelector(
  selectProducts,
  products => products.length
);

export const selectAvailableProductCount = createSelector(
  selectAvailableProducts,
  products => products.length
);
```

### Pattern 3: View model selector
```typescript
// src/app/products/state/products.selectors.ts
export interface ProductsListViewModel {
  products: Product[];
  loading: boolean;
  error: string | null;
  totalCount: number;
}

export const selectProductsListViewModel = createSelector(
  selectAvailableProducts,
  selectLoading,
  selectError,
  selectProductCount,
  (products, loading, error, totalCount): ProductsListViewModel => ({
    products,
    loading,
    error,
    totalCount,
  })
);
```

### Pattern 4: Dictionary-based selector (no projector)
```typescript
// src/app/products/state/products.selectors.ts
export const selectProductsListViewModel = createSelector({
  products: selectAvailableProducts,
  loading: selectLoading,
  error: selectError,
  totalCount: selectProductCount,
});
```

### Pattern 5: Factory (parameterized) selectors
```typescript
// src/app/products/state/products.selectors.ts
export const selectProductById = (id: string) =>
  createSelector(selectProducts, products => products.find(p => p.id === id) ?? null);
```

Cached factory selector to preserve memoization across calls:
```typescript
// src/app/products/state/products.selectors.ts
const selectorCache = new Map<string, MemoizedSelector<object, Product | null>>();

export const selectProductById = (id: string) => {
  if (!selectorCache.has(id)) {
    selectorCache.set(
      id,
      createSelector(selectProducts, products => products.find(p => p.id === id) ?? null)
    );
  }
  return selectorCache.get(id)!;
};
```

### Pattern 6: Cross-feature selectors
```typescript
// src/app/shared/state/cross-feature.selectors.ts
import { selectCartItems } from '../../cart/state/cart.selectors';
import { selectProducts } from '../../products/state/products.selectors';

export const selectCartWithDetails = createSelector(
  selectCartItems,
  selectProducts,
  (cartItems, products) =>
    cartItems.map(item => ({
      ...item,
      product: products.find(p => p.id === item.productId)!,
      subtotal: item.quantity * (products.find(p => p.id === item.productId)?.price ?? 0),
    }))
);
```

### Pattern 7: Custom memoization with deep equality
```typescript
// src/app/shared/state/custom-selectors.ts
import { createSelectorFactory, defaultMemoize } from '@ngrx/store';

const isDeepEqual = (a: unknown, b: unknown): boolean =>
  JSON.stringify(a) === JSON.stringify(b);

export const createDeepEqualSelector = createSelectorFactory(
  (projectionFn: (...args: any[]) => any) =>
    defaultMemoize(projectionFn, undefined, isDeepEqual)
);
```

### Pattern 8: Using selectSignal in components
```typescript
// src/app/products/products-list.component.ts
import { Component, inject } from '@angular/core';
import { Store } from '@ngrx/store';
import { selectProductsListViewModel } from './state/products.selectors';

@Component({
  selector: 'app-products-list',
  template: `
    @if (vm().loading) {
      <app-spinner />
    } @else if (vm().error; as error) {
      <app-error [message]="error" />
    } @else {
      <p>{{ vm().totalCount }} products found</p>
      @for (product of vm().products; track product.id) {
        <app-product-card [product]="product" />
      }
    }
  `,
})
export class ProductsListComponent {
  private readonly store = inject(Store);
  readonly vm = this.store.selectSignal(selectProductsListViewModel);
}
```

### Pattern 9: selectSignal with custom equality
```typescript
// src/app/products/products-list.component.ts
readonly products = this.store.selectSignal(selectProducts, {
  equal: (a, b) => a.length === b.length && a.every((item, i) => item.id === b[i].id),
});
```

### Pattern 10: Using extraSelectors with createFeature (bridges Ch 8)
```typescript
// src/app/products/state/products.feature.ts
import { createFeature, createSelector } from '@ngrx/store';
import { productsReducer } from './products.reducer';

export const productsFeature = createFeature({
  name: 'products',
  reducer: productsReducer,
  extraSelectors: ({
    selectProducts,
    selectLoading,
    selectError,
    selectQuery,
  }) => {
    const selectFilteredProducts = createSelector(
      selectProducts,
      selectQuery,
      (products, query) =>
        query
          ? products.filter(p =>
              p.name.toLowerCase().includes(query.toLowerCase())
            )
          : products
    );

    return {
      selectFilteredProducts,
      selectProductsListViewModel: createSelector({
        products: selectFilteredProducts,
        loading: selectLoading,
        error: selectError,
      }),
    };
  },
});
```

## Breaking Changes and Gotchas

- **No breaking changes to selector APIs in NgRx v21.** The selectors API is stable and unchanged from v20.
- **Selectors with props deprecated since v11.** `MemoizedSelectorWithProps` still exists in the type system but should not be used. Factory selectors are the replacement.
- **Two-generic `createFeatureSelector<AppState, FeatureState>()` deprecated.** Use `createFeatureSelector<FeatureState>(featureName)`.
- **`defaultMemoize` uses `===` (reference equality).** Returning new arrays/objects from projectors (`.map()`, `.filter()`, spread) creates new references every time, defeating downstream memoization even if values are identical.
- **Factory selectors create new instances per call.** Each invocation of a factory selector creates a fresh selector with its own memoization cache. For performance-sensitive scenarios, cache selector instances in a `Map`.
- **`selectSignal` vs `toSignal(store.select(...))`:** `selectSignal` is preferred because it handles equality checking correctly and is more concise. `toSignal` requires `{ requireSync: true }` or an initial value, and does not apply result-level equality by default.
- **`release()` in tests:** Call `selector.release()` between test cases to clear memoized state and avoid test pollution.

## Common Mistakes and How to Fix Them

1. **Using a broad feature selector as input when only one property is needed.** Fix: use granular child selectors as inputs.
2. **Storing derived state in the store alongside raw data.** Fix: compute derived values in selectors.
3. **Using `combineLatest` in components to combine `store.select()` calls.** Fix: compose a single selector using `createSelector`.
4. **Putting transformation logic in template-bound methods.** Fix: move logic into selectors for memoization.
5. **Multiple separate `selectSignal` calls instead of a view model selector.** Fix: create one view model selector returning all needed data.
6. **Factory selectors in `@for` loops without caching.** Fix: cache created selectors in a `Map` or compute outside the loop.
7. **Ignoring `release()` in long-lived apps.** Fix: call `release()` when selectors are no longer needed to free memory.
